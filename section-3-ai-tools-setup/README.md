# Section 3: AI 도구 환경 구성

> **소요 시간**: ~1시간 | 이 섹션의 흐름: AI Gateway (비용 통제) → Claude Code 사용 → AI Dev Kit 설치 → Genie Code 연결
>
> **참고**: 이 워크샵은 **Track C (Genie Code)** 전용입니다. Section 3은 Genie Code의 기능을 확장하는 AI Dev Kit 구성에 초점을 맞추며, Claude Code는 **"이런 것도 가능하다"는 참고용 소개**입니다. 직접 설치하지 않아도 워크샵 진행에 문제 없습니다.

---

## 이 섹션에서 배우는 것

```
Step 1: AI Gateway — 비용을 통제할 수 있다
         ↓
Step 2: Claude Code — 이 환경에서 안전하게 사용할 수 있다
         ↓
Step 3: AI Dev Kit — 설치하고 Databricks에 연결한다
         ↓
Step 4: Genie Code + MCP — 노트북 안에서도 확장된 기능을 쓴다
```

| 단계 | 배우는 것 | 핵심 메시지 |
|------|----------|-----------|
| [AI Gateway](ai-gateway.md) | 비용 관리, Rate Limit, 모니터링 | "AI 도구를 비용 통제하면서 도입할 수 있다" |
| [Claude Code 사용하기](claude-code.md) | 설치, AI Gateway 연결, 사용법 | "이 환경에서 Claude Code를 안전하게 쓸 수 있다" |
| [AI Dev Kit 설치](ai-dev-kit-setup.md) | MCP 서버 배포, Skills 배포 | "AI Dev Kit으로 Databricks 기능을 확장한다" |
| [Genie Code + AI Dev Kit](genie-code-aidevkit.md) | MCP 연결, 도구 선택, 테스트 | "Genie Code에서 크로스 프로덕트 작업이 가능해진다" |

# Genie Code란?

**Genie Code**는 Databricks 플랫폼 안에 내장된 AI 코딩 어시스턴트입니다. 일반적인 AI 코딩 도구(ChatGPT, Copilot)와 달리, Databricks 환경의 **테이블 구조, 데이터, 실행 환경**을 직접 이해하고 코드를 실행합니다.

## 일반 코딩 AI와의 핵심 차이

| | ChatGPT / Copilot | **Genie Code** |
|---|---|---|
| 테이블 스키마 | 모름 (사용자가 알려줘야) | **Unity Catalog에서 자동 읽음** |
| 코드 실행 | 안 됨 (생성만) | **직접 실행 + 결과 확인 + 에러 자동 수정** |
| 데이터 탐색 | 안 됨 | **테이블 검색, 샘플 조회, 프로파일링** |
| 작업 범위 | 코드 한 조각 | **계획 → 실행 → 검증 → 수정 자동 반복** |
| 환각 | 컬럼명 지어냄 | **실제 메타데이터 기반, 환각 최소화** |

![Genie Code 사이드 패널](images/genie-code-panel.png)
*Genie Code 사이드 패널 — Agent 모드에서 자연어로 프롬프트를 입력하면 코드를 생성하고 실행합니다*

## 어디서 쓸 수 있나?

Genie Code는 Databricks의 **6개 제품 영역**에서 동작합니다:

| 제품 영역 | 주요 용도 | 호출 방법 |
|----------|----------|----------|
| **노트북** | EDA, 코드 생성, 모델 학습 | `Cmd+I` 또는 사이드 패널 |
| **SQL Editor** | SQL 작성, 최적화 | `Cmd+I` 또는 사이드 패널 |
| **AI/BI 대시보드** | 대시보드 생성/수정 | 사이드 패널 |
| **Lakeflow Pipeline Editor** | SDP 파이프라인 코드 생성 | 사이드 패널 |
| **MLflow** | GenAI 앱 디버깅, 트레이스 분석 | 사이드 패널 |
| **Jobs** | 에러 진단, 코드 수정 | 사이드 패널 |

> 📸 **[스크린샷]**: 각 제품 영역에서 Genie Code 아이콘이 보이는 위치 (6개 캡처)

## 비용 구조

Genie Code의 비용은 **AI 영역**과 **컴퓨트 영역**으로 나뉩니다:

| 영역 | 포함 항목 | 비용 |
|------|---------|------|
| **Genie Code AI 영역** | 프롬프트 입력, 컨텍스트 수집(UC 메타데이터), LLM 추론(코드/쿼리 생성) | **무료** (LLM 비용은 Databricks가 부담) |
| **컴퓨트 영역** | 생성된 코드/쿼리를 사용자의 기존 컴퓨트에서 실행 | **기존 컴퓨트 사용료** |

> 컴퓨트 선택 우선순위: 현재 attached 클러스터 → 마지막 실행 클러스터 → 기본 SQL Warehouse → Serverless.
> Genie Code가 별도 컴퓨트를 생성하지 않으며, 사용자가 원래 쓰던 컴퓨트에서 코드를 실행할 뿐입니다.

### 무료 작업 vs 컴퓨트 과금 작업

| 구분 | 작업 예시 | 비용 |
|------|---------|------|
| 코드 생성/수정 | 자동완성, 인라인 제안, `/explain`, `/doc`, `/fix`, `/optimize` | **무료** |
| 코드 리팩토링 | 노트북 코드 구조 변경, 변수명 수정 제안 | **무료** |
| 자료 조사/설명 | 에러 분석, 코드 설명, 최적화 제안 | **무료** |
| 데이터 조회 | SELECT 쿼리 실행, 테이블 탐색, `/findTables` | 컴퓨트 과금 |
| Agent Mode 실행 | 코드 셀 실행, 검증 단계 | 컴퓨트 과금 |
| 대시보드/EDA | 시각화 생성 시 데이터 조회 | 컴퓨트 과금 |

> **핵심**: "코드를 생성하는 것"은 LLM이 하므로 무료, "코드를 실행하는 것"은 컴퓨트가 하므로 과금.

> 💡 **동작 방식**: 프롬프트를 보내면 Genie Code가 자동으로 Serverless 컴퓨트를 켭니다. 코드 실행이 끝난 뒤 15분간 추가 작업이 없으면 자동으로 꺼집니다. Chat Mode에서 "이 코드 설명해줘"처럼 실행이 필요 없는 질문은 컴퓨트를 켜지 않으므로 비용이 발생하지 않습니다.

## 내부 아키텍처

Genie Code는 Multi-Agent 아키텍처로 동작합니다:

| 레이어 | 구성 요소 | 역할 |
|--------|---------|------|
| **Genie Code Agent** | Task Planner → Model Router → Tool Executor | 사용자 프롬프트를 계획하고, 최적 모델을 선택하고, 도구를 실행 |
| **Foundation Models** | Azure OpenAI GPT + Anthropic Claude + Custom Models | Databricks가 태스크별 최적 모델을 자동 선택 (사용자 선택 불가) |
| **내장 도구 (~50개)** | 코드 실행, 데이터 조회, 자산 관리, SQL 쿼리, MCP 연동 | Agent가 목적에 맞는 도구를 자동 호출 |
| **Unity Catalog 컨텍스트** | 테이블 스키마, 컬럼 설명, 데이터 리니지, 권한 정보, 태그, 샘플 데이터 | 코드 생성 시 실제 메타데이터를 참조하여 환각 최소화 |

### LLM 모델 상세

| 항목 | 내용 |
|------|------|
| 주요 모델 | Azure OpenAI GPT (Azure 호스팅) + Anthropic Claude (AWS 호스팅) |
| 모델 선택 | Databricks가 태스크별 최적 모델 자동 선택 (고객 선택 불가) |
| 데이터 보호 | **Zero Data Retention** — 사용자 데이터로 모델 학습 없음 |
| 전송 암호화 | TLS 암호화 |
| 저장 암호화 | AES-256 |
| 콘텐츠 필터링 | Azure OpenAI 콘텐츠 필터 + Anthropic 안전 메커니즘 적용 |

## Chat Mode vs Agent Mode

| 구분 | Chat Mode | Agent Mode |
|------|-----------|------------|
| 동작 방식 | 인라인 코드 제안/자동완성 | 자율적 계획→코드→실행→검증→수정 |
| 사용자 개입 | 모든 코드를 사용자가 실행 | 코드 실행 전 확인 요청 (approve/allow) |
| 적합한 작업 | 빠른 코드 완성, 설명, 수정 | 복잡한 EDA, 파이프라인 구축, 디버깅 |
| 컨텍스트 | 현재 셀/파일 중심 | 전체 노트북 + UC 메타데이터 활용 |
| Slash 명령 | `/explain`, `/fix`, `/doc` | 자연어 대화로 복합 작업 수행 |

> 이 핸즈온에서는 **Agent Mode**만 사용합니다.

### 처음 시작할 때 — 첫 번째 프롬프트

Genie Code를 처음 열었다면, 아래 프롬프트로 환경이 제대로 되어 있는지 확인해보세요:

```
안녕, 나는 Databricks를 처음 써. 
지금 내가 접근할 수 있는 카탈로그와 스키마 목록을 보여줘.
```

> 이 프롬프트를 Agent Mode에서 실행하면, Genie Code가 `SHOW CATALOGS`와 `SHOW SCHEMAS` 쿼리를 자동으로 만들어 실행합니다.

> 💡 **팁**: 개발/테스트 중에는 `LIMIT 1000`을 습관적으로 붙이면 컴퓨트 비용을 줄일 수 있습니다.
